# 训练过程

## 第一次

- 时间：4月8号~4月11号

- 结果：失败

- 现象：

  - 加载 checkpoints 验证模型训练情况时发现产生了推理坍塌。测试使用不同的 prompt 输入，模型的输出都被固定的高频 token 占据，高频 token 有很高的 logits。目标 token 的 logits 远小于哪些高频 token，排行也十分靠后（几百名之外）。
  - 最初通过计算 hidden states 和 logits 之间的余弦相似度发现，hidden states 的结果能够区分，但是 logits 结果都趋近于1。当时初步判断是 backbone 学到了内容但是 lm_head 退化导致的。但忽略了一个关键点：**余弦只衡量方向，不衡量幅度**。
  - hidden states 余弦结果看起来有差异，是因为余弦归一化掉了DC分量的幅度优势，差异方向得以明显。但是实际推理使用的是点积，DC 分量 94.6% 的模长优势在点积计算里被完整保留，导致 lm_head 中和 DC分量角度只要是正相关的 token 分数就很大，直接主导了 logits 结果。

- 原因：
  1. optimizer 在设置 weight decay 时没有对参数进行分组，全部参数都进行统一的衰减。
  2. 训练数据过于单一，Edu 4-5 分子集高度同质，教科书式连接词（在、是、以、对、通过）在语料中极度高频，高频 token 的分布过度集中。
  3. **学习率偏高（5e-4）**，模型在激进的学习率下为了更快的降低 loss 找到捷径。
  4. untied 只是在这些错误叠加的情况下加速了 hidden states 的退化。

- 解决方案：
  1. 降低 lr，全局 lr 设置为 2e-4，lm_head 的 lr 单独设置为 4e-5。
  2. 对不同模块设置不同的权重衰减，主干矩阵正常衰减；norm、embeddings 不进行权重衰减；lm_head 进行权重衰减，并可以适当调高。
  3. 添加数据多样性，添加Edu中2-4分的数据集和中文维基百科数据。

- 错误链路

  ```text
  Edu 4-5 分数据文体单一 + 学习率 5e-4
          ↓
  模型找到捷径：表征偏向高频 token 方向可快速降 loss
          ↓
  DC 分量持续膨胀，占模长 94.6%
  （奇异值第一方向占比 82%）
          ↓
  点积计算 logits 时，DC 分量的巨大模长主导结果
  高频 token logit 值被放大，其他 token 信号被压制
          ↓
  所有 prompt 的 logits 余弦 ~0.97
  
  模型无法区分上下文，推理坍塌
  ```
