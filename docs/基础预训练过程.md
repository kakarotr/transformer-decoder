# 训练过程

## 第一次

- 时间：4月8号~4月11号
- 结果：失败
- 现象：加载 checkpoints 验证模型训练情况时发现产生了推理坍塌。测试使用不同的 prompt 输入，模型的输出都被固定的高频 token 占据，高频 token 有很高的 logits。目标 token 的 logits 远小于哪些高频 token，排行也十分靠后（几百名之外）。通过验证不同 prompt 下模型的 last hidden state 和 logits 的 cosine 结果得出，last hidden state 存在差异，但是 logits 相似度趋近于1。这说明 backbone 已经学到了上下文的差异，但是 lm_head 在进行线性投影时上下文的差异被抵消了，导致最后输出的 token 分布高度一致（先验分布）。
- 原因：
  1. 独立 lm_head 在这次训练下学成了过强的共享先验读出器，虽然 embeddings 和 lm_head 没有权重绑定，但是这并不表示原因是 untied。更可能是 untied + 当前训练参数 + 当前数据分布组合导致的错误。
  2. optimizer 在设置 weight decay 时没有对参数进行分组，全部参数都进行统一的衰减。
  3. 训练数据中存在大量的网页信息和非中文信息，即使是高质量的中文语料，也存在混合分布。
- 解决方案：
  1. 对 optimizer 进行参数分组。
     - 主干矩阵：正常 lr，正常 weight decay。
     - norm、embeddings：正常 lr，weight decay=0
     - lm_head：weight decay=0，单独更低 lr
  2. 降低学习率：
     - 基础学习率：3e-4
     - lm_head：1e-4 ~ 1.5e-4
  3. 提高语料中文纯度
     - 提高连续语料的中文占比
     - 减低英文、代码、网页模板痕迹
