# 项目背景

本项目的目标，是从零开始完整实践一个大语言模型的训练流程，包括：模型定义、数据收集与清洗、分词器构建、预训练、持续预训练以及监督微调（SFT）。项目重点不在于复现通用大模型的全部能力，而在于在**可控的训练资源条件下**，独立完成一个垂直领域中文语言模型的全链路实现与验证。

## 项目目标

在**中文语境**下围绕**垂直领域**进行问答，目前候选领域为**日本战国**和**第二次世界大战**（二选一，或分别独立训练）。经过 SFT 后，模型应能按照指定的**Markdown 格式**生成结构化回答，例如标题、分点列表、分级小节和重点强调等，以提升回答的可读性与可控性。

## 训练阶段总览

训练流程分为三个阶段，其中**基础预训练产出的模型基座是唯一的**，后续所有垂直领域的持续预训练均在此基座上独立展开，互不干扰。

```
基础预训练（通用中文基座）
│
├──▶ 持续预训练（日本战国）──▶ SFT（日本战国）
│
└──▶ 持续预训练（第二次世界大战）──▶ SFT（第二次世界大战）
```

## 训练阶段详述

### 阶段一：基础预训练

使用通用中文语料训练模型的基础语言能力，产出**唯一的通用模型基座**，
供后续任意垂直领域复用。该阶段同步训练**通用分词器**，不包含任何垂直领域内容。

**数据集：**
- `HuggingFaceFW/fineweb`：使用其中中文部分，清洗后约 1.04 GB。
- `openbmb/Ultra-FineWeb`：使用其中中文部分且 `score > 0.9` 的数据，清洗后约 40 GB。

**当前状态：** 预训练数据清洗已完成，正在准备训练基础词表。

---

### 阶段二：持续预训练（以领域为单位，独立进行）

在基础模型基座上，针对具体垂直领域进行持续预训练，补充领域知识、
术语、人物关系、事件脉络及相关表达方式。

每个领域的持续预训练流程独立，包括：
1. 在领域数据上训练**领域专用分词器**，并将领域 token 追加到基础分词器中。
2. 在基座模型上使用领域数据进行持续预训练。

**领域数据保留简单的 Markdown 结构标记**，使模型在学习领域知识的同时，
逐步适应结构化文本分布。

**数据来源（以日本战国为例）：**
中文维基、日文维基（大模型汉化）、网站文章、Z-Library 相关书籍。

**当前状态：** 尚未开始，等待基础预训练完成。

---

### 阶段三：监督微调（SFT）

在对应领域的持续预训练模型基础上进行 SFT，目标是让模型稳定地按预设格式
回答用户问题，强化指令跟随能力、信息组织能力和 Markdown 输出规范性。

**当前状态：** 尚未开始，数据集待构建，等待持续预训练完成后推进。

---

## 模型定义

使用 **PyTorch** 从零实现 **Decoder-Only Transformer** 架构，采用主流设计，
覆盖实际训练所需的核心组件，不依赖现成大模型框架。目的是深入理解模型结构、
训练流程和工程细节的关系，保持可解释、可修改、可扩展的基础。

**总参数量：** 0.78B（暂定）

> 注：具体超参数（层数、d_model、注意力头数、上下文长度、位置编码、FFN 类型等）
> 待分词器构建完成后确认并补充。

## 技术栈

| 项目         | 选型                                    |
|--------------|-----------------------------------------|
| 框架         | PyTorch                                 |
| 分布式策略   | DDP（DistributedDataParallel）          |
| 混合精度     | BF16                                    |
| 优化器       | AdamW                                   |
| 学习率调度   | Linear Warmup + Cosine Decay            |
| 分词器库     | Hugging Face Tokenizers                 |
| 训练监控     | `rich` 控制台输出 + 逐步指标写入文件   |

## 评估方案

当前阶段采用 **eval loss** 作为主要评估指标，并通过 `math.exp(eval_loss)`换算为**困惑度（Perplexity）**进行直观比较。

SFT 阶段的评估方案（如格式遵循率、领域问答质量等）待后续设计。
