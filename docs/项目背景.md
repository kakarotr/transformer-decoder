# 项目背景

本项目的目标，是从零开始完整实践一个大语言模型的训练流程，包括：模型定义、数据收集与清洗、分词器构建、预训练、持续预训练以及监督微调（SFT）。项目重点不在于复现通用大模型的全部能力，而在于在**可控的训练资源条件下**，独立完成一个垂直领域中文语言模型的全链路实现与验证。

模型的最终目标，是在**中文语境**下围绕**日本战国**这一垂直领域进行问答，并在 SFT 后能够按照指定的 **Markdown 格式**生成结构化回答，例如标题、分点列表、分级小节和重点强调等，以提升回答的可读性与可控性。

在模型实现方面，项目使用 **PyTorch** 从零实现 **Decoder-Only Transformer** 架构，并尽量采用当前主流语言模型中的常见设计，尽可能覆盖实际训练所需的核心组件，而不依赖现成的大模型框架直接拼装。这样做的目的，是深入理解模型结构设计、训练流程和工程细节之间的关系，并在后续迭代中具备可解释、可修改、可扩展的基础。

在训练资源方面，项目当前以**个人电脑**作为主要实验与验证环境。现有本地环境为 **Windows 11 + RTX 5070 Ti（16GB 显存）**，因此当前阶段的模型规模、数据配比、训练策略和实现复杂度都需要围绕单机训练条件进行权衡。现阶段本地实验规划的模型参数规模约为 **0.6B**，主要用于验证模型实现、训练流程和数据方案的可行性。若本地训练结果达到预期，后续将进一步迁移至**云平台上显存更大的 GPU**，尝试对更大参数规模的模型继续训练与扩展。

在数据规划方面，项目分为三个阶段：

1. **基础预训练阶段**
  使用 Hugging Face 上的 `opencsg/Fineweb-Edu-Chinese-V2.2` 数据集，并选择其中 `score=4~5` 的高质量中文数据（大小为70G），作为模型通用中文能力与基础语言建模能力的主要来源。

2. **持续预训练阶段**
  使用与**日本战国**相关的垂直领域数据，帮助模型补充领域知识、术语、人物关系、事件脉络以及相关表达方式。该部分数据目前保留了简单的 **Markdown 结构标记**，以便模型在继续学习领域知识的同时，逐步适应结构化文本分布。

3. **监督微调（SFT）阶段**
  目标是让模型学会按照预设格式稳定回答用户问题，强化指令跟随能力、信息组织能力和 Markdown 输出规范性。该阶段数据目前尚未构建，后续将围绕“问题—标准答案”对进行设计，并重点覆盖日本战国领域的常见问答场景。

总体来说，这是一个以**工程实践**和**垂直领域建模**为核心的项目：一方面验证从零实现 Transformer Decoder 并完成训练闭环的可行性，另一方面探索在有限算力条件下，如何先完成本地小规模验证，再逐步扩展到更大训练资源与更大模型规模，最终构建一个具备基础问答能力、领域知识能力以及结构化输出能力的中文语言模型。
