# 模型结构说明

## 1. 模型概述

本项目当前实现的是一个基于 **PyTorch** 的 **Decoder-Only Transformer** 语言模型，用于中文语境下的自回归文本生成任务。模型整体采用现代主流语言模型中的常见设计，并围绕后续预训练、持续预训练和监督微调（SFT）进行实现。

模型的总体结构可以分为三层：

1. **基础组件层（components）**
   包括注意力机制、前馈网络、位置编码和归一化模块等，用于构成每一层 Transformer Block。

2. **主干网络层（Decoder）**
   由多个 `DecoderLayer` 堆叠而成，负责对输入序列进行逐层特征提取与上下文建模。

3. **语言建模层（Causal Language Model）**
   在 `Decoder` 输出的基础上，通过线性映射将隐藏状态投影到词表维度，得到每个位置的 token logits，用于训练和生成。

---

## 2. 文件说明

- `attention.py`：实现多头注意力机制，是 `DecoderLayer` 中的核心模块之一。
- `mlp.py`：实现前馈网络模块，当前使用的是适合现代语言模型的激活与投影结构。
- `rope.py`：实现旋转位置编码（RoPE），用于为注意力机制提供相对位置信息。
- `rms_norm.py`：实现 RMSNorm，用于替代传统 LayerNorm。
- `decoder.py`：包含 `DecoderLayer` 和 `Decoder` 表示单层结构和主干网络。
- `causal_lm.py`：最终语言模型封装。

## 3. 模块层级关系

```
CausalLanguageModel
├── Decoder
│   ├── Token Embedding
│   ├── N × DecoderLayer
│   └── Final RMSNorm
└── lm_head
```

每个 `DecoderLayer` 内部又由以下结构组成：

```
DecoderLayer
├── RMSNorm
├── Multi-Head Attention
├── Residual Connection
├── RMSNorm
├── MLP
└── Residual Connection
```

整体上采用标准的 **Pre-Norm** 结构，即先归一化，再进入子模块，最后加残差。

## 4. 各模块职责说明

### 4.1 **CausalLanguageModel**

`CausalLanguageModel` 是最外层的语言模型封装模块，负责：

- 接收输入 token ids
- 调用 `Decoder` 得到隐藏状态
- 通过 `lm_head` 将隐藏状态映射到词表大小
- 输出用于训练或推理的 logits

该模块本质上是整个模型的任务接口，负责把 Transformer 主干连接到语言建模目标上。

---

### 4.2 **Decoder**

`Decoder` 是模型的主干网络，负责完成从离散 token 到上下文化表示的变换。

其主要组成包括：

- **Embedding 层**

  将输入 token ids 映射为连续向量表示。

- **多层 DecoderLayer 堆叠**

  逐层提取上下文信息，建模序列内部依赖关系。

- **最终归一化层**

  对最后一层输出进行归一化，提升训练稳定性。

`Decoder` 不直接输出词表概率，而是输出隐藏状态，供上层语言建模头使用。

---

### 4.3 **DecoderLayer**

`DecoderLayer` 是 Transformer 主干中的基本重复单元，负责单层上下文建模。

每层通常包含两个子模块：

1. **自注意力模块**

   负责让每个位置与其历史位置进行信息交互，从而建模上下文依赖。

2. **前馈网络模块（MLP）**

   负责对每个位置的表示进行独立的非线性变换，增强模型表达能力。

此外，每个子模块前都带有归一化层，并在子模块输出后加上残差连接，以保证深层训练的稳定性。

---

### 4.4 **Multi-Head Attention**

注意力模块负责计算序列中不同位置之间的依赖关系，是 Transformer 最核心的组件之一。

当前实现的职责包括：

- 对输入隐藏状态进行线性投影，得到 Query、Key、Value
- 使用 RoPE 对 Query 和 Key 注入位置信息
- 结合因果掩码（Causal Mask）和Padding掩码（Padding Mask）限制当前位置只能关注自身及之前的位置
- 计算注意力分数并加权聚合 Value
- 将多个头的输出拼接后映射回隐藏维度

若后续支持 KV Cache，则该模块还将承担推理阶段缓存历史 Key/Value 的职责。

### 4.5 **MLP**

MLP 模块负责对注意力输出做进一步的特征变换。

其作用包括：

- 对每个位置的隐藏状态进行通道方向上的投影和非线性变换
- 增强模型表达能力
- 补充注意力机制无法直接提供的逐位置变换能力

当前实现采用 **SwiGLU** 风格的前馈结构，以提升参数利用效率和表达能力。

---

### 4.6 **RoPE**

RoPE（Rotary Position Embedding）用于为注意力机制注入位置信息。

与绝对位置编码不同，RoPE 通过对 Query 和 Key 进行旋转变换，使模型在注意力计算中自然感知相对位置信息。该方法目前在主流 Decoder-Only 模型中广泛使用。

在本项目中，RoPE 由独立模块实现，并在注意力模块内部调用。

---

### 4.7 **RMSNorm**

RMSNorm 是当前模型中使用的归一化方式，用于提升训练稳定性。

相较于 LayerNorm，RMSNorm 在结构上更简洁，且已被多个现代大语言模型采用。当前模型在每个 `DecoderLayer` 中使用 RMSNorm，并采用 Pre-Norm 形式组织网络结构。

## 5. 关键设计选择

当前模型的主要设计选择包括：

- **Decoder-Only 架构**
   适用于自回归语言建模任务，也是当前主流大语言模型的常见方案。
- **Pre-Norm 结构**
   有利于提升深层网络训练稳定性。
- **RMSNorm**
   替代传统 LayerNorm，简化结构并对齐现代语言模型常见实现。
- **RoPE 位置编码**
   用于建模序列中的相对位置信息。
- **模块化实现**
   将注意力、MLP、RoPE、归一化等核心组件独立拆分，便于测试、调试与后续扩展。

## 6. 总结

当前模型以 **Decoder-Only Transformer** 为核心，采用模块化方式实现注意力、前馈网络、位置编码和归一化组件，并通过 `DecoderLayer -> Decoder -> CausalLanguageModel` 的层级组织形成完整语言模型。该实现既服务于后续预训练、持续预训练和 SFT，也为进一步理解现代语言模型的结构细节、训练逻辑和工程实现提供了基础。
